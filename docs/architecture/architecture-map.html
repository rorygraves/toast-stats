Workspaces

Four npm workspaces. Dependencies flow upward: shared-contracts is the base, analytics-core is the engine, collector-cli is the orchestrator, frontend is the consumer.

▸ @toastmasters/shared-contracts
packages/shared-contracts
base

Cross-workspace Zod schemas + derived TypeScript types. Dual ESM/CJS build.

Consumers: analytics-core, collector-cli, frontend

Deps: zod

▸ @toastmasters/analytics-core
packages/analytics-core (~58K LOC)
pure compute

Pure TypeScript computation engine. No I/O. Tested at 85%+ coverage.

Key modules: AnalyticsComputer, BordaCountRankingCalculator, DistinguishedDistrictCalculator (TI Item 1490), CompetitiveAwardsCalculator, ClubHealthAnalyticsModule, MembershipAnalyticsModule, +9 more

▸ @toastmasters/collector-cli
packages/collector-cli
CLI / pipeline

4-stage pipeline CLI (commander).

  • scrape — Playwright → GCS raw-csv/{date}/
  • transform — CSV → typed DistrictStatistics
  • compute — analytics-core → JSON
  • upload — gsutil cp → GCS bucket

Logging: stderr only (R4); stdout reserved for structured JSON

Data flow (primary path)

Daily, idempotent. Snapshots are immutable; re-running for the same date overwrites cleanly.

# Actor Action Output
1 GitHub Actions (cron) Trigger collector-cli for yesterday Playwright session
2 collector-cli scrape Fetch CSV exports from dashboards.toastmasters.org gs://…-staging/raw-csv/{date}/
3 collector-cli transform csv-parse → validate with Zod from shared-contracts typed DistrictStatistics
4 collector-cli compute AnalyticsComputer runs ~14 modules (Borda, DD, ClubHealth, …) AnalyticsComputationResult per district
5 collector-cli upload gsutil cp snapshots + indices → staging v1/latest.json, v1/dates.json, snapshots/{date}/*.json
6 GitHub Actions (promotion) gsutil cp staging → production bucket gs://toast-stats-data-ca/
7 Cloud CDN Serve from production bucket cdn.taverns.red/v1/*.json
8 Frontend (TanStack Query) fetchCdnManifest → fetchCdnRankings → fetchDistrictAnalytics User sees fresh data
⚠ Key invariants
  • Snapshots are immutable per date. Re-running 'daily' for the same date is idempotent.
  • Pipeline + frontend share Zod schemas via shared-contracts — contract drift is caught at build time.
  • TanStack 5-min stale window coexists with CDN 1-hour cache; effective freshness is bounded by both.
  • All paths in collector-cli MUST call validateDistrictId() before path.join() (Lesson 01 / CodeQL #57).

Architecture decisions

Distilled from CLAUDE.md, sprint history, and the 60+ lessons. See architecture.json for the full set.

▸ ADR-001 · Compute once, serve static

Daily pipeline pre-computes ALL analytics into immutable JSON; frontend reads from CDN. Trade-off: schema changes need a pipeline backfill — not a hot deploy.

▸ ADR-002 · Shared Zod schemas, dual ESM+CJS

shared-contracts owns the wire format. Pipeline + frontend cannot drift; types verified at build time.

▸ ADR-004 · Year-cumulative metrics have base = 0 each PY

Distinguished Clubs, Officer Awards, etc. reset July 1. No new base field needed — Δ equals current. (Lesson 57)

▸ ADR-005 · % Distinguished uses Paid Club Base, not Active Clubs

Per TI Item 1490. The District 93 production bug (#537) caught this — count thresholds correctly used base while the trophy case used current. (Lesson 60)

▸ ADR-006 · Pin related queries to the same snapshot date

Two independent "latest" resolutions can drift across a CDN publish. Thread the resolved date through every related query. (Lesson 59)

▸ ADR-007 · TDD with separate RED and GREEN commits

RED → GREEN never combined. After pre-push hook was removed, separation makes test-after slips visible in commit history. (Lesson 54)

▸ ADR-008 · WCAG 2.4.7 focus-within rings for invisible overlays

When stacking opacity-0 native controls over visible chrome, add focus-within:ring-* on the parent so keyboard users see focus. (Lesson 58)

⚠ Active tripwires + known issues

From CLAUDE.md tripwires + open GitHub issues. Read these BEFORE touching the relevant subsystem.

Tripwire
SnapshotBuilder.build() has TWO district-tracking code paths

Success path + validation-failure path. Both must be updated when changing tracking logic.

Tripwire
DCP goals are INDEPENDENT, not sequential

Use clubPerformance raw fields. Never infer count as Goals 1 → N.

Tripwire
Chart || 1 range fallback causes y-axis inversion

Pad symmetrically when range === 0 instead.

Security tripwire
path.join() with raw user input = path traversal

ALWAYS call validateDistrictId() / resolvePathUnderBase() first. Lesson 01 / CodeQL #57.

Open issue #525
Integration journey tests fail ~20% under CI parallel load

Pass in serial; v8 coverage instrumentation contention. Root-cause likely renderWithProviders bloat (Lesson 53).

Open issue #488
Lighthouse CLS = 0.217 on / fails threshold

Re-runs mask it intermittently. Real layout-shift bug in initial paint.

Open issue #482
Pre-push hook was removed

Local TDD discipline now depends on agent enforcement. A sustainable carved-out gate is still needed.

Open issue #522
D## chip + bare numeric districtName pattern duplicated 4 places

Same #520 bug shape exists on RegionPage, search-suggestions, CommandPalette. Extract <DistrictChipAndName>.
