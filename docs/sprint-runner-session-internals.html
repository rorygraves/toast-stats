Deep dive · Inside Stage 6 (Work)

Inside the Spawned Claude Session

The overview diagram shows the runner's ten lifecycle stages. This page zooms into the one stage where the autonomous work actually happens — inside the spawned claude --remote-control session. It answers three questions:

① How does the session get started? Process tree + what each layer passes to the next.
② What does the session know? Context scopes layered from system prompt to current file edits.
③ What skills run and when? Activation triggers + which context layers each skill consumes.

Click any element below to see its role, contents, and the surrounding context. Theme toggle top-right.

§1

Process stack & spawn messaging

~5 forks, 3 message types

From launchd to the Anthropic API, the spawned session sits at the bottom of a five-level process tree. Each layer hands something specific to the next: env vars (launchd → script), shell command + cwd (script → screen), argv with prompt (bash → claude), and HTTPS stream (claude ↔ Anthropic + claude.ai). Click each row for the exact handoff.

└ /sbin/launchd macOS init · PID 1
↓ env: META_EPIC=606, STRICT_GATE=1, PATH, HOME
└ scripts/sprint-runner.sh orchestrator · ~700 LOC
↓ fork: screen -dmS sprint-runner-618 bash -c "..."
└ SCREEN -dmS sprint-runner-618 detached PTY · macOS screen daemon
↓ spawn: login -pflq + bash -c
└ bash -c "cd $WORKTREE && claude ..." inner shell · cwd = sprint-618 worktree
↓ exec: claude --remote-control sprint-618 "$(cat $PROMPT_FILE)"
└ claude --remote-control sprint-618 the agent · interactive UI in detached PTY
↓ HTTPS: prompt cached, tool calls, model responses
└ api.anthropic.com (Claude Opus 4.7) model inference · prompt-cached
↑ parallel: pair connection from claude.ai/code
└ claude.ai Remote Control (sprint-618) operator-side pairing · monitor + interrupt
Sibling processes spawned by the claude session itself (children of the agent, not shown in the stack above): git, gh, npm, playwright, curl, gsutil, and any other CLI invoked by tool calls. Each runs as a short-lived child of the claude PID inside the worktree's cwd.
§2

Context scopes

6 layers, outermost loaded once at session start

The claude session reads context from six progressively narrower scopes. Outer scopes load once at session boot and stay resident; inner scopes load on demand via tool calls or skill invocations. Click any layer for what it contains and which skills consume it.

L1 · User-global CLAUDE.md

~/.claude/CLAUDE.md

Operator's engineering manifesto. TDD discipline, no-shortcuts rules, commit cadence, persona triggers. Same across every project.

Scientific rigor Issue-driven dev DoD checklist Persona triggers

L2 · Project CLAUDE.md

toast-stats/CLAUDE.md

Project-specific rules: tripwires, conventions, paths, key dependencies. Engineering Rules (R1–R11) live here.

Tripwires R1–R11 rules npm workspaces Brand v1.0 notes

L3 · Auto-memory

memory/MEMORY.md + linked files

Persistent across sessions. User profile, feedback patterns, project state references. Linked memory files load on demand by topic.

User profile Feedback patterns Data pipeline ref Brand v1.0 ref

L4 · Bootstrap prompt (first user turn)

scripts/sprint-bootstrap.prompt

The runner's template, substituted with this sprint's variables. Tells the session: re-read rules + 5 newest lessons, re-confirm predecessor + prod, invoke /sprint, then close-out in strict order.

{{EPIC}} = 617 {{SPRINT_N}} = 1 {{TARGET_ISSUE}} = 618 {{PREV_ISSUE}} = none 5-step closeout

L5 · Skills (loaded on demand)

~/.claude/skills/ + project skills/

Each SKILL.md is loaded only when the skill activates (Skill tool, user invocation, or auto-trigger). Brings persona-specific rules + workflow.

/sprint /ron-pm /ron-sa /ron-ux /ron-qe /verify /run /review

L6 · Working context (per turn)

Tool results · file reads · current diff

The innermost, most volatile layer. Loaded by tool calls during the session: Read a file, Grep a pattern, Bash output, GitHub API responses. Trimmed/compacted as context fills.

Read tool results gh issue view payloads git diff output test runner stdout Playwright screenshots

§4

Message flow during one sprint

14 numbered turns · cradle to PR-merge to closeout

Sequential turns inside the spawned session, from bootstrap-prompt arrival through epic-closure. Each step shows the actor pair (caller → callee) and the message intent. Click any step for surface details.

bash inner shell → claude binary

Cold start: claude --remote-control sprint-618 "<bootstrap prompt text>"

Bootstrap prompt baked into argv. Session reads it as the first user turn. Working dir = sprint-618 worktree.

claude → Read tool

Re-read rules: tasks/rules.md + 5 newest tasks/lessons/*

Per bootstrap step 1. Loads project-specific conventions and the most recent learnings. Re-grounds even if the session was running pre-cached.

claude → Bash tool · gh CLI

Re-confirm predecessor + prod reachability

gh issue view {{PREV_ISSUE}} + curl -sfI https://ts.taverns.red. Belt-and-suspenders against runner-side staleness.

claude → Bash tool · gh CLI

Re-read epic body: confirm Sprint N (#{{TARGET_ISSUE}}) still target

gh issue view {{EPIC}} --json body. Trust the epic over the bootstrap prompt — operator may have advanced something.

claude → Skill tool · /sprint

Invoke /sprint with target #{{TARGET_ISSUE}}

/sprint loads its SKILL.md, brings sprint-execution framework into L5. From this point /sprint drives the Red → Green → Refactor cycle.

/sprint → Skill tool · /ron-* persona(s)

Activate work-appropriate persona(s) based on issue body

UI work → /ron-ux. Architecture change → /ron-sa. Each persona's SKILL.md merges into L5. Multiple personas can be active simultaneously.

/sprint → Read/Grep/Bash

RED — write failing test for the new behaviour

Inspect existing code, write the test, run it. Confirm it fails for the right reason. Commit with #{{TARGET_ISSUE}} ref.

/sprint → Edit/Write/Bash

GREEN — minimal implementation to pass the test

Smallest change that turns red to green. Re-run test, confirm pass. Commit.

/sprint → Edit/Bash

REFACTOR — clean while keeping tests green

Extract helpers, rename, tighten types. Full suite still green. Commit.

/sprint → Bash · git + gh

Open PR, watch CI, merge on green

git push -u + gh pr create + watch checks. If CI red, report + exit.

/sprint → /verify skill

Live verify on ts.taverns.red after deploy

Playwright smoke + CDN snapshot data-path check. Capture screenshots. SendUserFile for operator visibility.

claude → Bash · gh CLI (4 calls)

Closeout sequence (strict order per #626)

Post evidence comment → gh issue edit --add-label sprint-verified → PATCH epic body to tick checkbox → gh issue close sub-issue.

claude → Bash · grep + gh

If last sprint, close epic too (#636 step 5)

grep -c '^- \[ \] \*\*Sprint' epic body. Zero remaining? Post completion summary + gh issue close the epic.

claude → exit

Exit. Inner bash runs echo; screen terminates naturally.

Worktree stays on disk (cleaned up next runner tick via gc_worktrees). Auto-branch may or may not still exist depending on merge config.
