# p1 — Extract QUERY_CLIENT_CONFIG and build the client from it

## Summary

In `frontend/src/config/queryClient.ts`, introduce an exported typed
config constant and build the existing `queryClient` from it, with zero
behaviour change. Add one vitest test asserting the config and the
client's default options.

## Scope

Touch exactly two files:

- `frontend/src/config/queryClient.ts` (modified)
- `frontend/src/config/queryClient.test.ts` (new)

Nothing else changes. Do not create `cacheTimes.ts`. Do not edit any file
under `frontend/src/hooks/` or `frontend/src/components/`.

## Required changes

### `frontend/src/config/queryClient.ts`

1. Add an **exported named TypeScript interface** describing the config
   shape (the five fields below).
2. Add an **exported `const QUERY_CLIENT_CONFIG`** of that interface type
   with exactly these fields and values:
   - `queryRetry = 1`
   - `refetchOnWindowFocus = false`
   - `staleTimeMs = 5 * 60 * 1000`
   - `gcTimeMs = 10 * 60 * 1000`
   - `mutationRetry = 0`
3. Build the existing exported `queryClient` from `QUERY_CLIENT_CONFIG`,
   mapping the fields onto `defaultOptions.queries`
   (`retry`, `refetchOnWindowFocus`, `staleTime`, `gcTime`) and
   `defaultOptions.mutations` (`retry`). The `QueryClient` constructor
   must contain **no inline numeric literals** — every value comes from
   `QUERY_CLIENT_CONFIG`.

### `frontend/src/config/queryClient.test.ts` (new vitest file)

- Assert each of the five `QUERY_CLIENT_CONFIG` field values
  (`queryRetry === 1`, `refetchOnWindowFocus === false`,
  `staleTimeMs === 5 * 60 * 1000`, `gcTimeMs === 10 * 60 * 1000`,
  `mutationRetry === 0`).
- Assert that `queryClient.getDefaultOptions()` reflects them — i.e. the
  resolved query/mutation default options match the corresponding
  `QUERY_CLIENT_CONFIG` fields.

## Acceptance criteria

1. `queryClient.ts` exports a named interface for the config shape and an
   exported `const QUERY_CLIENT_CONFIG` of that type with fields
   `queryRetry`, `refetchOnWindowFocus`, `staleTimeMs`, `gcTimeMs`,
   `mutationRetry` set to `1`, `false`, `5 * 60 * 1000`, `10 * 60 * 1000`,
   `0` respectively.
2. The `queryClient` is constructed from `QUERY_CLIENT_CONFIG`; the
   `QueryClient` constructor body contains no inline numeric literals.
3. All five values are byte-identical to the previous implementation —
   zero behaviour change.
4. `frontend/src/config/queryClient.test.ts` exists, asserts the five
   `QUERY_CLIENT_CONFIG` values, and asserts that
   `queryClient.getDefaultOptions()` reflects them. It passes under
   `npm run test:frontend`.
5. No file outside the two listed above is modified; `cacheTimes.ts` is
   not created.
6. **Every string literal in new or changed code, including imports, must
   use double-quote characters.**
7. `npm run build:frontend` (typecheck + build) passes.
